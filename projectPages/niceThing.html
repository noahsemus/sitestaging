renais­sance

renaissance was a semester long project for my web design class. the first half of the semester consisted of user research and wireframing, with the second having students delve into visual design. the first half of the semester is thoroughly contained and annotated in a ux deck, which can be viewed below.

the second half of the semester was focused on visual design and prototyping. i used the wireframes and ux as more of a guide rather than a bible, and attempted to verge off it as i developed the visual language more.

thought process: the spectrum interaction as noted in the ux deck was changed in the visual design. having users interact with the main element of the site proved to be a problematic idea during user research. it was changed to a bottom slide with the use of 3D cards to showcase hero media right on the homescreen.

the library cards, much like everything else on the site, is colour coded to where in the spectrum the user lies.

each card expands to show the relevant services the user has connected within.

deeper level recommendation pages tell users how they identify the users' taste. a refute action is included to customize the assignments. the top of the hero recommendation card peeks out from the bottom to tell the user scrolling is possible.

the feed was created as another way users can discover new content. they are able to see what their friends are watching as well as what colour they are. this is the reasoning for the card design -- as it calls out the colour and the content before the actual content of the post itself.

the profile generates ai descriptions of what each user likes within each media category. here there was another opportunity for content discovery by placing links to relevant media related to the description on the vertical list to the right.

profile history allows users to view where they were on the spectrum at different points in time. here they can view various spectrum locations and the content in such sections.

renaissance was built to be responsive, as such two prototypes were created. here the mobile prototype is shown. the mobile site is meant to act and load like an app via its connected animations. this would be done through google's instant apps api.

the final prototype, of course, showcases the entire site itself. mainly, the spectrum interaction on the homepage is shown -- as the 3d card effects promote user delight and exploration.
